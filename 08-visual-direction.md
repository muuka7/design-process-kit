# Visual Direction

## Purpose

Visual direction should exist as a separate artifact after the core structure is known and before implementation polish begins.

It is not a substitute for wireframes or page contracts.

## Define

- reference products
- what to copy
- what to avoid
- emotional register
- typography direction
- spacing and density
- color personality
- motion principles
- mobile vs web visual differences

## Gate

You should be able to look at a proposed design and quickly decide whether it matches the intended visual system.

## Warning

Do not let visual direction replace information hierarchy.
If the wireframe is weak, a visual style guide will not fix the product.
