# Contributing

## Purpose

This repository is a public design-process kit for wireframe-first, prototype-first product design work.

Contributions should keep the kit:

- generic
- reusable
- framework-aware but not project-bound
- safe to publish publicly

## Good Contributions

- improve workflow clarity
- add reusable templates
- improve greenfield and brownfield guidance
- add examples that are clearly generic
- fix portability issues such as broken relative links
- improve README and contribution ergonomics

## Avoid

- project-specific screenshots
- private company or product details
- internal URLs, local filesystem paths, or environment-specific instructions
- hardcoded app/domain examples that only fit one business
- implementation details that assume a specific private codebase

## Repo Conventions

- Prefer relative links in docs.
- Keep examples neutral and reusable.
- Keep the kit usable for both web and mobile JavaScript/TypeScript teams.
- If you add a new stage or template, update `README.md` so the flow remains coherent.

## Pull Requests

Please make pull requests:

- small
- focused
- easy to review

When relevant, explain:

- what gap the change fixes
- whether it affects greenfield, brownfield, or both
- whether README or template updates are also needed

## Safety Rule

If a contribution makes the kit less portable or introduces project-specific material, it should not be merged without a strong reason.
