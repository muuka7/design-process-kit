# Greenfield Discovery

Use this when the app or feature area does not exist yet.

## Discovery Goals

- define the product mental model
- define information architecture
- define the product job
- define target users
- define platform scope
- confirm data-contract sources before screen design becomes too detailed

## Required Inputs

- PRD or product brief
- user roles
- platform scope
- backend or API documentation
- schema docs, OpenAPI docs, database schema, or sample payloads

## If Backend Docs Exist

- map domains and entities first
- identify the minimum viable screen inventory
- design wireframes against likely data reality

## If Backend Docs Do Not Exist

- call out the risk immediately
- create provisional page contracts with explicit assumptions
- keep backend fields and sample values marked as `assumed`

## Deliverables

- idea capture
- mental model
- information architecture
- product intake
- screen inventory
- low-fidelity wireframes
- prototype brief
- page contracts with explicit data assumptions
